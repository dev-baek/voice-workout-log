# S02 운동 기록 앱 화면 디자인 스펙

## 문서 상태

- 문서 번호: `s02`
- 문서 타입: frontend
- 대상 기능: 운동 기록 앱형 녹음 홈 화면
- 현재 단계: Phase 0 UI 개선
- 마지막 업데이트: 2026-06-30

## 목적

기존 녹음 PoC 화면을 테스트 도구가 아니라 실제 운동 기록 웹앱의 첫 화면처럼 보이게 만든다.

이번 화면은 로그인 없이 동작한다. 사용자는 앱을 열고 바로 운동 내용을 말로 기록할 수 있어야 한다.

## 화면 구성

```txt
앱 헤더
-> 녹음 액션 패널
-> 주간 요약
-> 최근 기록
-> 부위별 볼륨
```

## 디자인 방향

- 흰색 기반의 조용한 운동 기록 앱 UI로 둔다.
- 강조색은 파란색 하나만 주로 쓰되 화면 전체가 단색으로 보이지 않게 회색, 초록, 빨강 상태색을 보조로 사용한다.
- 마케팅 hero가 아니라 실제 앱 첫 화면처럼 구성한다.
- 카드는 개별 정보 단위에만 쓰고 중첩 카드는 만들지 않는다.
- 카드 radius는 8px 이하로 둔다.
- 사용자가 가장 먼저 눌러야 하는 것은 `녹음 시작`이다.

## 주요 영역

### 앱 헤더

- 앱 이름: `말하는 운동기록`
- 보조 문구: `오늘 운동을 말로 남겨보세요.`

### 녹음 액션

- 현재 상태 문구를 보여준다.
- `녹음 시작`, `녹음 중지` 버튼을 제공한다.
- 녹음 완료 후 오디오 미리듣기와 녹음 길이를 보여준다.

### 대시보드 미리보기

- 이번 주 운동 횟수
- 총 세트 수
- 총 볼륨

### 최근 기록

데모 데이터로 최근 운동 기록을 보여준다.

- 벤치프레스
- 스쿼트
- 랫풀다운

### 부위별 볼륨

가슴, 하체, 등 기준의 간단한 막대 그래프를 보여준다.

## 이번 범위에서 안 하는 것

- 로그인
- 서버 업로드
- STT/AI 구조화
- IndexedDB 저장
- 실제 통계 계산
- 라우팅 추가
